# Процессы (краткая пользовательская инструкция)

Виджет `Процессы` автоматизирует работу с amoCRM: запускается по выбранному событию, проверяет условия и последовательно выполняет настроенные действия.

## 1. Что подготовить заранее

- Активное подключение amoCRM.
- Синхронизированные поля, воронки, этапы, пользователи и другие справочники amoCRM.
- Понимание, какое событие должно запускать процесс и какие действия необходимо выполнить.

На странице списка процессов доступны:

- `Подключить amoCRM` — подключение аккаунта amoCRM.
- управление вебхуками amoCRM — установка, проверка и удаление;
- `Исполнения процессов` — общая история всех запусков.

## 2. Создание процесса

1. Откройте `Процессы`.
2. Нажмите `Создать процесс`.
3. Укажите название процесса.
4. Выберите триггер.
5. Добавьте действия и условия.
6. Сохраните и включите процесс.

Кнопка `+` между блоками добавляет новый шаг в выбранное место.

## 3. Триггеры

Процесс может запускаться:

- вручную;
- по расписанию;
- относительно даты;
- по событию amoCRM: создание, изменение, удаление, смена статуса и другим;
- из другого процесса.

Для активных событий amoCRM платформа устанавливает необходимые вебхуки.

Для запуска дочернего процесса:

1. Выберите у дочернего процесса триггер `Запуск из другого процесса`.
2. Добавьте в родительский процесс действие `Запустить процесс`.
3. Выберите в действии созданный дочерний процесс.

Дочерний процесс нельзя включить, пока он не подключён к реальному действию родительского процесса.

## 4. Действия и условия

Основные доступные действия:

- создать или изменить сделку, контакт либо компанию;
- изменить статус сделки;
- поставить задачу или добавить примечание;
- найти сущность;
- запустить SalesBot;
- отправить уведомление по email или Telegram;
- запустить другой процесс.

Условие направляет выполнение по ветке `Да` или `Нет`. Для сравнения можно использовать готовые значения, собственный текст и переменные процесса.

## 5. Переменные и автоматический выбор сущности

Переменные позволяют использовать данные триггера и результаты предыдущих шагов:

```text
{{lead.id}}
{{contact.id}}
{{lead.cf(123456)}}
{{proc_1_found_1}}
```

Кнопка `Переменные` открывает справочник переменных и ID полей, воронок, этапов и групп пользователей.

Если ID сущности в действии не указан вручную, процесс пытается использовать подходящую сущность из результатов предыдущих шагов или данных триггера.

## 6. Проверка и запуск

Кнопка `Запустить тест` выполняет процесс на реальных данных.

Важно: во время теста действия действительно отправляются в amoCRM и подключённые сервисы. Перед запуском проверьте настройки шагов и выбранные сущности.

После проверки включите процесс кнопкой в списке процессов.

## 7. Контроль исполнения

Страница `Исполнения процессов` показывает все запуски процессов:

- процесс и триггер;
- статус и количество выполненных шагов;
- время запуска и длительность;
- ошибку при наведении на красную иконку статуса;
- возможность перезапустить завершённое исполнение.

Нажмите на строку, чтобы посмотреть выполненные шаги и связанные сущности amoCRM.

## 8. Карта связей

Кнопка `Карта связей` показывает:

- триггер процесса;
- действия и условия;
- ветки `Да` и `Нет`;
- запущенные дочерние процессы и их действия.

Из карты можно перейти к редактированию связанного процесса.

## 9. Рекомендованный порядок работы

1. Подключить amoCRM и обновить справочники.
2. Создать процесс и выбрать триггер.
3. Добавить действия и условия.
4. Проверить переменные и ID сущностей.
5. Запустить тест.
6. Проверить результат в `Исполнения процессов`.
7. Включить процесс.
