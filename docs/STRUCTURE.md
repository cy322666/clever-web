# Repository structure

```
.
├── 1.html
├── README.md
├── application
│   ├── app
│   │   ├── Console
│   │   ├── Exceptions
│   │   ├── Filament
│   │   ├── Helpers
│   │   ├── Http
│   │   │   ├── Controllers
│   │   │   └── Middleware
│   │   ├── Jobs
│   │   ├── Livewire
│   │   ├── Models
│   │   ├── Observers
│   │   ├── Providers
│   │   ├── Requests
│   │   └── Services
│   ├── bootstrap
│   ├── config
│   ├── database
│   │   ├── factories
│   │   ├── migrations
│   │   └── seeders
│   ├── public
│   ├── resources
│   │   ├── css
│   │   ├── js
│   │   ├── lang
│   │   └── views
│   ├── routes
│   ├── storage
│   ├── tests
│   └── vendor (generated by Composer)
├── docker
│   └── images
│       ├── php82
│       ├── postgres
│       └── supervisor
└── docker-compose.yml
```

## Notes
- The Laravel application lives in `application/`, which follows the standard Laravel directory layout (controllers, models, jobs, services, etc.).
- Docker images are stored in `docker/images/` and are orchestrated by the root `docker-compose.yml`.
